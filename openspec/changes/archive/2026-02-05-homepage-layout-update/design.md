## 上下文

当前首页 `src/pages/index.astro` 使用传统的列表布局，只显示标题和日期。我们从 `blog.leowang.net` 导入了新文章，这些文章都包含高质量的特色图片（heroImage）。为了更好地展示这些内容，提升用户体验，我们需要重构首页布局。

## 目标 / 非目标

**目标：**
- 将首页文章列表转换为“左图右文”的卡片式布局。
- 响应式设计：在移动端，布局应自动调整为“上图下文”。
- 图片比例协调，建议使用 16:9 或 3:2，并设置 `object-cover` 防止变形。
- 文本区域包含：可点击的标题（H2/H3）、简短摘要（截取前N个字或使用 description）、作者（固定或从 frontmatter 获取）、发布日期。

**非目标：**
- 分页功能（本次变更仅关注布局样式）。
- 侧边栏小部件（保持单栏流式布局）。

## 决策

1.  **CSS 框架**：
    -   **选择**：Tailwind CSS。
    -   **理由**：项目已有依赖，Flexbox/Grid 实现布局非常简便。

2.  **图片处理**：
    -   **选择**：使用 HTML `<img>` 标签，配合 Astro 的 `Image` 组件（如果可用）或直接引用 `/images/...` 路径。
    -   **理由**：为了简化，先直接使用 Markdown frontmatter 中的 `heroImage` 路径。如果 `heroImage` 缺失，则不渲染图片区域，退化为纯文本宽卡片。

3.  **摘要生成**：
    -   **选择**：优先使用 frontmatter 中的 `description`。
    -   **理由**：导入的文章已包含 description，无需运行时截取正文内容。

## 风险 / 权衡

-   **风险**：老文章没有 `heroImage`。
    -   *缓解*：设置条件渲染，仅当 `heroImage` 存在时显示图片列。或者提供一个默认的 fallback 图片。
    -   *决定*：为保持整洁，若无图片，该文章卡片仅显示文本，但保持宽度一致。

## 迁移计划

直接修改 `src/pages/index.astro`。这是一个展示层变更，不涉及数据结构迁移。
