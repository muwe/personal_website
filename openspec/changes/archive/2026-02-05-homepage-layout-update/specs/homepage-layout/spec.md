## 新增需求

### 需求：首页文章卡片布局
首页必须以图文结合的卡片形式展示文章列表。

#### 场景：桌面端展示
- **当** 用户在宽屏设备（大于 768px）访问首页
- **那么** 每篇文章应显示为一个横向卡片
- **并且** 左侧为文章特色图片（heroImage），宽度占比约 40%-50%
- **并且** 右侧为文章信息，包含标题、摘要、作者和日期

#### 场景：移动端响应式展示
- **当** 用户在移动设备（小于 768px）访问首页
- **那么** 文章卡片应调整为垂直布局
- **并且** 特色图片在上方，占满宽度
- **并且** 文本信息在图片下方

#### 场景：图片缺失处理
- **当** 文章没有定义 `heroImage`
- **那么** 卡片应仅显示文本部分
- **并且** 文本区域应自动扩展占满卡片宽度

#### 场景：内容元素展示
- **当** 渲染文章卡片
- **那么** 标题应加粗并链接到文章详情页
- **并且** 摘要应显示 `description` 字段的内容，限制行数（如最多3行）
- **并且** 日期应格式化显示（如 "Feb 05, 2026"）
