## 新增需求

### 需求：项目基础架构
系统必须基于 Astro 框架初始化，并集成 Tailwind CSS 进行样式管理。

#### 场景：开发环境启动
- **当** 开发者运行开发服务器启动命令
- **那么** 系统应在本地端口启动，并正确加载 Astro 和 Tailwind 配置

### 需求：排版与主题系统
系统必须定义全局的设计 Token，包括字体栈（serif/sans）、字号阶梯、颜色变量（支持亮色/深色模式）。必须针对中文阅读体验优化行高（layout rhythm）。

#### 场景：默认字体渲染
- **当** 用户访问任何页面
- **那么** 页面正文应优先使用配置的衬线/无衬线字体栈，确保清晰易读

#### 场景：深色模式切换
- **当** 用户系统设置为深色模式或手动切换主题
- **那么** 网站颜色应平滑过渡到深色配色方案，且对比度符合 WCAG AA 标准

### 需求：响应式布局框架
系统必须实现响应式布局，包括统一的页眉（Header）、页脚（Footer）和主要内容区域（Main）。

#### 场景：移动端访问
- **当** 视口宽度小于 768px
- **那么** 布局应自动调整间距，导航菜单应适配移动端显示
